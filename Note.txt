- Git Clone : clone source về local

- Git Branch : xem branch

- Git log --oneline : xem history

- Git status

- Git commit

- Git push

- Git Stash:
Trong trường hợp bạn chuyển sang nhánh mới khi đang code dang dở, thì sẽ bị yêu cầu commit code, tuy nhiên không phải lúc nào bạn cũng muốn commit code cả.
Có một cách là sẽ đẩy toàn bộ working tree hiện tại vào stash bằng lệnh git stash. Sau đó khi muốn lấy lại working tree đang làm thì dùng lệnh git stash pop để lấy ra.
git stash
git stash pop

- Git Diff:
git diff master develop
git diff master develop --name-only

- Git pull: Cập nhật local branch với thay đổi ở remote branch
Khi hai người cùng làm việc trên một nhánh, việc người này phải cập nhật code mới của người kia là điều bình thường. Để làm việc này dùng lệnh git pull
git pull origin develop

- Tạo mới nhánh tại một commit cụ thể
Trường hợp muốn tạo một nhánh có code tương ứng với một commit nào đó thì sẽ tạo ra 1 nhánh mới với HEAD trỏ tới commit hoặc một tag (git tag) cụ thể nào đó
git checkout -b [branch_name] [commit_id]

- Git delete : xóa nhánh
git branch -d [branch_name]
Tham số -d ám chỉ việc xóa nhánh khi nhánh đó đã đồng bộ với nhánh upstream tương ứng hoặc đã merge thành công vào HEAD hiện tại
Nếu muốn xóa bằng mọi giá thì dùng tham số -D (git sẽ xóa nhánh bất chấp bất cứ điều gì xảy ra)

- Phục hồi branch khi đã xóa
Mở lịch sử reflog : git reflog
Tìm commit mới nhất liên quan đến việc xóa 
Phục hồi nhánh bằng cách checkout đến commit đã chọn: git checkout -b [branch_name] [commit_id]
